# /리서치 — 맥락 인식 리서치 에이전트

자연어 요청 하나로 다중 도구 병렬 리서치를 수행한다.
**핵심 원칙**: 시작 전 의문은 모두 해소, 시작했으면 끝까지.

## 사용법

```
/리서치 [자연어 요청]

예시:
/리서치 댄싱컵 6월 SNS 캠페인용 글감, MZ세대 펫 트렌드
/리서치 CMKF 청년 멘토링 프로그램 해외 사례
/리서치 AI 마케팅 부트캠프 커리큘럼 벤치마크
/리서치 --맥락없이 [요청]      # 현재 프로젝트 맥락 무시, 순수 검색
/리서치 --맥락=댄싱컵 [요청]   # 다른 프로젝트 맥락 강제 로드
/리서치 --상세 [요청]          # 깊이 깊음 (Perplexity + exa + 1차 소스 다 파기)
/리서치 --빠름 [요청]          # 도구 2-3개만 (5분 내 응답)
```

---

## STEP 1. 맥락 자동 추출 (항상 실행, 토큰 가볍게)

다음을 순서대로 읽는다:

1. **cwd** — `pwd`로 현재 디렉토리. 클라이언트 폴더 패턴 매칭 (`1. CMKF/`, `2. 네이버 플레이스/`, `4. 휴닉/`, `6. 댄싱컵/`, `7. 브랜드 라이즈/`, `8. AI디지털위원회/`, `9. 레벨업/` 등)
2. **`{cwd}/CLAUDE.md`** — 있으면 읽기 (클라이언트 톤·타깃·작성규칙)
3. **`{cwd}/_PM/이어서.md`** — 있으면 핵심 추출 (최근 작업 흐름)
4. **최근 git log 5개** — `git log -5 --oneline` (있으면)
5. **요청 자체에 다른 클라이언트명이 섞여 있는지** 검사 — "댄싱컵", "CMKF" 등 키워드

추출 산출물:
```yaml
context:
  cwd: <현재 폴더>
  client: <자동 추론한 클라이언트, 없으면 "공통">
  client_tone: <CLAUDE.md에서 추출한 톤 1줄 요약>
  current_work: <_PM/이어서.md에서 추출한 진행 작업 1줄>
  recent_outputs: [<git log 5개 요약>]
request:
  raw: <사용자 원본 요청>
  intent_guess: <자연어에서 추론한 의도>
  scope_guess: {시간, 지역, 깊이, 소스유형}
```

추출 결과를 사용자에게 **3-5줄로 요약 출력**한다.

## STEP 2. 확인 게이트 (필수, AskUserQuestion 사용)

추출한 맥락을 바탕으로 **1~3개 질문**을 한 번에 1개씩 묻는다.
모든 차원이 명확하면 1개만, 모호한 차원이 많으면 최대 3개.

표준 차원 (이 중 모호한 것만 질문):

### Q1. 의도 확인 (가장 중요)
```
이 요청은 어디에 가장 가깝나요?
A) 트렌드 리서치 — 최근 흐름·시그널 발굴
B) 벤치마크 — 경쟁사·유사 사례 비교
C) 심층 조사 — 한 주제 깊게 파기
D) 사례 수집 — 다양한 사례 10+ 모으기
E) 도구·노하우 — 어떻게 하는지 방법론
F) 팩트체크 — 특정 정보 검증
```

### Q2. 범위 (시간 × 지역 × 깊이)
```
시간 범위 / 지역 / 깊이를 정해주세요.
A) 최근 1-3개월 / 한국 / 표준
B) 최근 1년 / 한국+해외 / 표준
C) 무관 / 해외 위주 / 깊음
D) 기타 (직접 입력)
```

### Q3. 산출물 형태
```
결과를 어디까지 저장할까요?
A) 콘솔 보고서만 (휘발성)
B) NotebookLM 노트북 생성 (장기 활용·재질의)
C) Notion 페이지 (팀 공유·참조)
D) A+B+C 모두
```

**규칙**:
- 한 번에 1개 질문, AskUserQuestion tool 사용
- 다중 선택형 우선
- 사용자가 "그냥 해줘" 류 답하면 추론값으로 진행
- 게이트 통과 후에는 **사용자 확인 없이 끝까지 진행** (STEP 6 박제 단계까지)

## STEP 3. 라우팅 (의도 × 도구 조합)

`~/.claude/research/routing.md` 참조. 핵심 라우팅 룰:

```
# 기본 의도 (Day-1)
trend + Korean + recent   → 네이버 Search + DataLab + Kakao/Daum + Perplexity(검증)
trend + overseas + recent → exa + Reddit + Perplexity(검증)
benchmark                  → exa + firecrawl-crawl(경쟁사) + Perplexity(검증)
deep dive                  → exa(deep) + firecrawl(1차 소스) + Perplexity(검증) + (학술 필요시) DBpia/KCI/RISS
case collection            → exa + firecrawl + Reddit + 큐레이션 소스 풀
tools/know-how             → context7 + WebSearch + Reddit
fact-check                 → Perplexity Sonar (전담) + 1차 소스 확인
SNS angle                  → SNS 세션 인프라 + firecrawl-instruct + 큐레이션 소스 풀
academic Korean            → DBpia + KCI + RISS + (보조)네이버 학술

# 5축 갭 보강 의도 (2026-05-22)
ad-reference        → Meta Ad Library + Google Ads Transparency + TikTok Creative Center + KIPRIS
                       (헬퍼: ~/.claude/research/scripts/meta-ad-library.md)
korean-community    → Naver Search + firecrawl-search(theqoo/fmkorea/dcinside/clien/pann/bobaedream/instiz)
                       + 썸트렌드 + (선택) Blind
trend-4point        → 네이버 데이터랩 + 썸트렌드 + 올영/무신사/29CM 랭킹 + BIGKinds (4축 채점)
influencer-korea    → 피처링 + Modash + TikTok One + (검증) HypeAuditor + 최근 협찬물 firecrawl
overseas-to-korea   → Product Hunt + Exploding Topics + Reddit Pro + Pinterest Trends
                       → (변환) → 네이버 데이터랩 + 썸트렌드 + 올영/무신사 한국 검증
```

**5축 갭 의도 호출 트리거**:
- "광고 레퍼런스 / 경쟁사 광고 / 메타 광고" → `ad-reference`
- "실제 후기 / 더쿠 / 디시 / 펨코 / 카피 훅 / 소비자 반응" → `korean-community`
- "트렌드 점수 / 검증해줘 / 이거 떠?" → `trend-4point`
- "인플루언서 / 크리에이터 캠페인" → `influencer-korea`
- "해외 트렌드 / Product Hunt / 한국 적용" → `overseas-to-korea`

**도구 역할 (기존 도구 재정의)**:
- `exa` — 해외 디스커버리 디폴트 (학술뿐 아니라 news/company/people 다 지원)
- `firecrawl` — 1차 소스 추출 (스크래핑·크롤·instruct 브라우저)
- `Perplexity Sonar` — Verifier 한정 (검색 발견 X, 인용 검증·합성 1회만)
- `NotebookLM` — YouTube/PDF/장기 합성·재질의용 박제
- `WebSearch` — Fallback (API 못 쓸 때만)
- `context7` — 라이브러리·SDK 문서 (도구·노하우 의도일 때만)

**큐레이션 소스 풀**: `~/.claude/research/sources.md` (마케팅·SNS 전문 사이트·뉴스레터·계정 목록)

**토큰 가지치기**:
- `--빠름` → 도구 2-3개만, 합성만
- 기본 → 도구 4-5개, 1차+2차
- `--상세` → 도구 6+, 1차+2차+교차검증+영상

## STEP 4. 병렬 실행

라우팅 결과로 정해진 도구 N개를 **단일 메시지 안에서 병렬 호출** (Promise.all 패턴).

```
예 (trend + Korean):
- 네이버 Search API (블로그·뉴스·카페 동시)
- 네이버 DataLab (트렌드 세그먼트)
- Kakao Daum Search
- firecrawl-search (큐레이션 소스 풀 우선)
- Perplexity Sonar (검증·인용)
```

API 키는 `~/.claude/research/.env`에서 로드 (없으면 사용자에게 발급 가이드 안내).

## STEP 5. 합성·교차검증·구조화

1. **중복 제거** — 같은 URL·같은 사실 1회만
2. **1차 소스 필터링** — SEO·리블로그·AI 요약 거르고 1차 소스 우선
3. **교차검증** — 2개 이상 도구에서 같은 사실 확인되는지 체크. 1개 소스만 있으면 `[단일 출처]` 표시
4. **구조화** — 의도별 다른 포맷
   - 트렌드/사례 → 표 (제목 / 1차 URL / 핵심 인사이트 / 클라이언트 톤 적용안)
   - 벤치마크 → 비교 표
   - 심층 → 섹션별 마크다운
   - 팩트체크 → 결론 + 근거 인용
   - **ad-reference** → 광고당 `훅 / 오퍼 / CTA / 랜딩` 4축 태깅 표 + 장기 운영★ 표시 + 패턴 분석 + 클라이언트 적용 훅 3개 (헬퍼: `~/.claude/research/scripts/meta-ad-library.md`)
   - **korean-community** → 4클러스터 `불만 / 칭찬 / 밈 / 표현` + 원문 인용 + URL
   - **trend-4point** → 4축 채점 표 (검색·소셜·커머스·뉴스) + 판정 (3↑실행/2관찰/1밈)
   - **influencer-korea** → `이름 / 플랫폼 / 팔로워 / 진성% / 브랜드핏 / 광고티 / 댓글반응 / 협찬빈도 / 가격대(추정)`
   - **overseas-to-korea** → 표 (해외 항목 / 변환안 / 한국 단계 0·시작·확산 / 우선순위)
5. **클라이언트 톤 반영** — 맥락 추출에서 얻은 톤(예: "MZ 펫 친근")으로 인사이트 한 줄 적용안 작성

## STEP 6. 박제 (사용자 답한 Q3 따라)

- **A 콘솔만** — 출력만 하고 끝
- **B NotebookLM** — `mcp__notebooklm-mcp__notebook_create` → 1차 소스 URL 일괄 `source_add` → 사용자가 나중에 질의 가능
- **C Notion** — `mcp__claude_ai_Notion__notion-create-pages`로 페이지 생성. 위치는 사용자 워크스페이스의 적절한 폴더 (HIZ 작업공간 / 9. 레벨업 / 리서치 박제)
- **D 모두** — A 출력 + B 노트북 + C 페이지

박제 후 한 줄로 위치 알려주고 종료.
