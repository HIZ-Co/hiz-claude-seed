현재 진행 중인 작업의 상태를 다음 세션에서 손실 없이 이어갈 수 있도록 `_PM/이어서.md`에 저장하세요.

> **2026-05-19 변경**: 옵시디언 vault PKM 자동 정리 섹션 제거. 회사 OS가 Notion+n8n+Slack+Claude Code+Drive로 피벗됨에 따라 vault 기반 정리는 더 이상 수행하지 않는다. 본 커맨드는 순수 `_PM/이어서.md` 저장만 담당.

## STEP 1. 작업 메모 저장 (항상 실행)

### 저장 위치
- 파일: `_PM/이어서.md` (현재 작업 디렉토리 기준)
- `_PM/` 폴더가 없으면 먼저 생성
- 기존 파일이 있으면 **덮어쓰기** (직전 체크포인트는 `_PM/이어서.이전.md`로 백업)

### 저장 내용 (이 형식 그대로)

```markdown
# 이어서 작업 메모

**저장 시각**: YYYY-MM-DD HH:MM
**작업 디렉토리**: <cwd>

## 1. 지금까지 한 일
- <완료된 작업 항목들 — 무엇을 왜 했는지>

## 2. 현재 상태
- <지금 어디까지 와 있는지 — 진행 중인 단계>
- <테스트/빌드/검토 결과가 있다면 그 결과>

## 3. 다음에 할 일
- [ ] <구체적인 다음 액션 1>
- [ ] <구체적인 다음 액션 2>
- [ ] <...>

## 4. 관련 파일·경로
- `<파일경로>` — <왜 중요한지>
- ...

## 5. 결정사항·맥락
- <이번 세션에서 사용자와 합의한 것, 방향 전환, 제외하기로 한 것>

## 6. 미해결·블로커
- <답을 받아야 할 질문, 막힌 부분, 외부 의존성>

## 7. 다음 세션 시작 프롬프트 제안
> <`/이어서불러오기` 후 사용자가 그대로 복붙해서 쓸 수 있는 한 문장>
```

### 작성 원칙
- **사실만**: 추측하지 말고 실제로 한 일·확인한 일만 기록
- **구체적으로**: "X 수정함"이 아니라 "X의 Y 함수에서 Z 부분 수정함"
- **다음 사람을 위해**: 이 파일만 보고도 맥락 없이 이어갈 수 있어야 함
- 누락이 의심되면 먼저 사용자에게 확인 후 저장

## STEP 1B. 페어 OS 워크로그 자동 적재 (남중 PC에서만 — 워크로그 레포 있을 때)

`_PM/이어서.md` 저장 직후, 이번 세션에서 한 일을 **페어 OS 학습 신호**로 워크로그에 적재한다.

### 조건
- `C:\Users\HIZ\worklog\` 폴더가 존재할 때만 수행. 없으면 이 단계 전체를 **조용히 건너뛴다**(다른 PC·환경 보호).

### 절차
1. **이번 세션 한 일을 요약** — STEP 1의 "1. 지금까지 한 일"을 토대로 워크로그 entry 작성.
2. **대상 파일**: `C:\Users\HIZ\worklog\logs\YYYY-MM-DD.md` (오늘 날짜).
   - 없으면 `C:\Users\HIZ\worklog\_template.md`를 복사해 생성(`YYYY-MM-DD`→오늘 날짜 치환).
   - 이미 있으면(같은 날 여러 세션) **덮어쓰지 말고 각 섹션에 항목 추가**.
3. **3개 섹션 채우기** (이 레벨만, 그 이하 디테일 금지):
   - `## 한 일 (활동)` — 무엇을 했는지
   - `## 어떻게 (기법·도구)` — 어떤 도구·방법·패턴으로
   - `## 배운 것 / 막힌 것 (인사이트)` — 배움·블로커
4. **commit + push**:
   ```
   cd C:\Users\HIZ\worklog
   git add logs/<오늘>.md
   git commit -m "worklog: <오늘> — <한 줄 요약>"
   git push
   ```

### 워크로그 민감정보 필터 (절대 위반 금지)
이 레포 내용은 **Anthropic API + Slack #green-automation**으로 흘러간다. 다음은 **워크로그에 절대 안 쓴다**(`_PM/이어서.md`에는 평소대로 기록 OK):
- ❌ 클라이언트 재무 수치·미공개 계약·회의 녹취 원문·인사/평가
- ❌ API 키·토큰·비밀번호·개인정보
- ❌ 사적 키워드(고민·이직·관계·감정 등)
- ✅ 활동·기법·도구·배움 레벨만. 클라이언트명은 일반 맥락 수준만(예: "CMKF 블로그 초안 3건").

판단 애매하면 그 항목은 워크로그에서 **생략**(빼는 쪽이 안전).

## STEP 2. 안전 장치 (절대 위반 금지)

1. **삭제 절대 금지** — `_PM/이어서.md`를 삭제하지 않는다. 항상 덮어쓰기 직전에 `_PM/이어서.이전.md`로 백업.
2. **클라이언트 폴더 보호** — `_PM/`는 cwd 하위에만 생성. 다른 프로젝트 폴더 침범 X.
3. **사적 키워드 보호** — 본문에 "고민·불안·이직·관계·감정" 등이 들어가면 그 부분만 요약하거나 사용자에게 확인 후 저장.

## 폐기된 기능 (2026-05-19)

다음은 이전 버전에서 수행하던 작업으로, 현재는 **수행하지 않는다**:

- ❌ 옵시디언 Vault 감지 (`_홈.md` + `Templates/` + `20-팀자산/`)
- ❌ vault 메타데이터 자동 보정 (YAML frontmatter)
- ❌ 인박스(`00-인박스/김남중/`) 자동 분류·이동
- ❌ 승격 후보 식별
- ❌ 묵은 노트·깨진 백링크·미완료 액션 스캔
- ❌ vault 통계 산출
- ❌ `_PM/이어서.md`에 PKM 정리 결과 추가 섹션

회사 OS = Notion+n8n+Slack+Claude Code+Drive (5차 세션 결정). 향후 박제 자동화는 n8n 워크플로우로 재구현 예정.
