이전 세션의 작업 상태를 읽고 이어서 진행하세요.

## 절차

1. **`_PM/이어서.md` 읽기** (현재 작업 디렉토리 기준)
   - 파일이 없으면: "이어서 메모가 없습니다. `_PM/이어서.md`를 먼저 만들어주세요." 라고 안내하고 중단
   - 파일이 있으면 전체를 읽기

2. **상태 요약 보고** (사용자에게)
   다음 형식으로 간결하게 보고:
   ```
   📌 이전 세션 요약
   - 작업: <한 줄 요약>
   - 마지막 상태: <어디까지 와 있는지>
   - 다음 할 일: <톱 1~3개>
   - 미해결: <블로커가 있다면>
   ```

3. **관련 파일 확인**
   - 메모에 언급된 핵심 파일들이 여전히 존재하는지 확인 (Read 또는 Glob)
   - 변경되었거나 사라진 파일이 있으면 사용자에게 알림

4. **이어서 진행**
   - 다음 액션이 명확하고 사용자가 추가 지시 없이 진행을 원하는 톤이면 그대로 시작
   - 모호하거나 우선순위가 갈리면 한 가지만 짧게 질문 후 진행
   - **확인 폭주 금지**: 단계마다 묻지 말고 끝까지 진행 (CLAUDE.md 규칙)

## 주의
- 메모는 **그 시점의 사실**입니다. 코드/파일이 그 사이 변경되었을 수 있으니, 실제 작업 전에 핵심 파일을 한 번 확인하세요.
- 메모와 현재 상태가 충돌하면 **현재 파일 상태**를 신뢰하고 사용자에게 차이를 알리세요.
